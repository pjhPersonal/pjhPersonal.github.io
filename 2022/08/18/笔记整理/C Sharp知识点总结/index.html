C#知识点总结

1. 值类型与引用类型

（1）值类型：bool, int, float, enum, struct…
（2）引用类型：string, object, class, delegate, interface…

string

这是一个值类型的赋值，但string是一个引用类型，string被分配在堆上，而不是栈上。

因此，当把一个字符串变量赋给另一个字符串变量时，会得到对内存中同一个字符串的两个引用。但是，string与其他引用类型在常见的操作上有一些区别。

例如字符串是不可改变的。修改其中一个字符串变量，就会创建一个全新的string对象，而另一个字符串变量不会发生任何变化。这实际上是运算符重载的结果。

数组

数组是引用类型，属于引用传递，用数组赋值新数组并不会产生新的副本。要想创建一个新的副本，使两个数组的数据互不干扰，需要用到Clone, CopyTo, 遍历数组赋值等方法。

2. 装箱与拆箱

（1）装箱：将值类型转换为object类型或由此值类型实现的任何接口类型的过程。（把值类型转换成引用类型）

  • 装箱是隐式的。

（2）拆箱：从对象中提取值类型。（把引用类型转换成值类型）

  • 拆箱是显示的。

3. 垃圾回收（GC）

（1）C#的GC只回收托管堆上的托管资源。

  • 非托管资源：stream（文件），connection（数据库连接），COM（组件）…
  • 非托管资源需要手动进行释放。

（2）GC步骤：

  1. 标记（Mark）：从Root开始进行引用标记，未被标记到的为不可达内存，不可达内存为GC对象。

  2. 重新分配地址（Relocate）：更新所有活动对象列表中的所有对象的引用，以便它们指向对象将在压缩阶段重定位到的新位置。

  3. 压缩（Compact）：当部分内存被清除后，原本的内存空间变得不连续，因此剩余的存活对象需要按照原始顺序从基址开始重新排列。

（3）非托管资源的释放：

  1. 终结器（以前称为析构函数）

    • 对象被GC回收之前被调用。
    • 无法在结构中定义终结器。 它们仅用于类。
    • 一个类只能有一个终结器。
    • 不能继承或重载终结器。
    • 不能手动调用终结器。
    • 会隐式调用对象基类上的Finalize方法，C# 编译器不允许重写该方法 Finalize。
    • 不应使用空终结器。 如果类包含终结器，会在Finalize队列中创建一个条目。 此队列由垃圾回收器处理。 当GC处理队列时，它会调用每个终结器。 不必要的终结器（包括空的终结器、仅调用基类终结器的终结器，或者仅调用条件性发出的方法的终结器）会导致不必要的性能损失。
    • GC运行时机不确定，可能导致非托管资源释放不及时。
    • GC.SuppressFinalize(object)： 不调用对象的终结器。

  2. 继承IDisposable接口，实现Dispose方法

  3. using关键字

The lifetime of a using local will extend to the end of the scope in which it is declared. The using locals will then be disposed in the reverse order in which they are declared.

4. 公共语言运行时（Common Language Runtime）

（1）CLR是微软为他们的.Net虚拟机所选用的名称，为微软的.Net产品提供运行环境。

（2）CLR上实际运行的并不是我们通常所用的编程语言，而是一种成为“通用中间语言”的字节码。这意味着只要能将代码编译成这种特定的“中间语言”，任何语言的产品都能运行在CLR上。

（3）.Net Framework是一个支持多种开发语言的开发平台，而这种多语言支持的特性以CLR为基础。CLR是一个.Net产品的运行环境。

（4）CLR的核心功能：内存管理，程序集加载，安全性，异常处理，线程同步等。

5. 静态构造函数

（1）静态构造函数用于初始化任何静态数据，或执行仅需执行一次的特定操作。 将在创建第一个实例或引用任何静态成员之前自动调用静态构造函数。

（2）执行顺序：静态变量 > 静态构造函数 > 实例变量 > 实例构造函数

6. 抽象类（abstract class）与接口（interface）

（1）相同点：

  1. 都可以被继承
  2. 都不能被实例化
  3. 都可以包含方法的声明

（2）不同点：

  1. 抽象类只能被单个类继承；接口可继承接口，并可多继承接口。
  2. 抽象基类可以定义字段、属性、方法实现；接口只能定义属性、索引器、事件、和方法声明，不能包含字段。
    字段：private int a = 0;
    属性：public int A {get;set;}
  3. 具体派生类必须覆盖（override）抽象基类的抽象方法；派生类必须实现接口的所有方法（没有override关键字）。
  4. 抽象类中的虚方法或抽象方法必须用public修饰；接口中的所有成员默认为public，不能有private修饰符。

7. 类（class）与结构体（struct）

（1）类：

  1. class属于引用类型，是分配在内存的堆上的。

（2）结构体：

  1. struct属于值类型，是分配在内存的栈上的。
  2. 不能从另外一个结构或者类继承，本身也不能被继承。（但是结构体可以实现一个或多个接口）
  3. 没有默认的构造函数，但是可以添加构造函数。
  4. 结构体中声明的字段无法赋予初始值。
  5. 可以不使用new初始化。这样必须初始化结构体内部所有的字段，否则在此之前字段都保持未赋值状态，且对象不可用。（就算使用new初始化，也是在栈上为其分配空间。）

8. 委托（delegate）与事件（event）

（1）委托：把一个方法作为参数传入另一个方法，本质上就是一个类，用来定义参数的类型。

  • 使用委托也可以将多个方法绑定到同一个委托变量，当调用此变量时可以依次调用所有绑定的方法。
  • public delegate void DelegateTypeName(Object paramName);

（2）事件：对委托类型变量的封装，类似于属性对字段的封装，目的是允许外部对委托变量进行注册的同时，避免外部对它进行随意的赋值等操作，保护对象的封装性。

  • 在类的内部，无论将事件变量声明成public还是protected，它本身总是private的（避免外部修改）。在外部对其进行注册（+=）和注销（-=）操作的访问限定符与事件变量声明时使用的访问符相同。
  • public event DelegateTypeName EventName;
  • 注册方法：
    1. aaa.EventName += bbb.funcB;
    2. aaa.EventName += (new B()).funcB; //用匿名对象注册
    3. aaa.EventName += new A.DelegateTypeName(bbb.funcB);
    4. aaa.EventName += C.FuncC; //注册静态方法

9. try, catch, finally, return

（1）无论如何，finally一定会执行。即使程序有异常，并且在catch中throw了，或者try/catch中有return，finally还是都会被执行。

（2）finally是在return后面的表达式运算完之后执行的。

（3）如果在finally中对返回值就行了重新赋值：

  1. 如果返回值是值类型，返回的值不受影响，即在try/catch时返回的值就已经确定了。（包括string类型，虽然是引用类型，这是特殊的个例）
  2. 如果返回值是引用类型，则会影响到返回值（e.g. 数组）。

（4）finally中不能有return，编译无法通过。

10. 值传递与引用传递

（1）值传递不是指参数是值类型，而是指形参变量会复制实参变量，也就是会在栈上多创建一个相同的变量。而按引用传递则不会。

（2）ref关键字：

  1. ref修饰参数：指示参数按引用传递。如果方法内部修改了参数，则会影响到方法外部。
    • 传递到ref或in形参的实参必须先经过初始化，然后才能传递。
    • ref, in或out不会影响方法重载。即如果方法签名仅ref, in或out不同，则不会通过编译。
  2. ref修饰返回值：“引用返回”，即让方法按引用向调用方返回，调用方可以修改返回的值。
    • 在签名中要用ref来标记返回值，在实现中的return语句中也要使用ref来标记返回值。
  3. ref修饰局部变量：用来指向return ref返回的值。
    • 无法将ref局部变量初始化为非ref返回值。
  4. ref修饰struct：告诉编译器它所修饰的struct只能被严格限制在栈上，不能通过任何方式放到堆上。（struct的装箱可能发生在例如将struct对象赋值/转换为一个object时）
    • ref struct的意义之一是用来实现Span
    • Span的定义就是一个ref struct，是一种允许表示任意内存的连续区域的值类型。（托管内存，非托管内存和栈内存都可以封装到Span中并安全地访问）

（3）in关键字：按引用传递参数，但是以只读的方式传递参数（确保参数在方法内不会被修改），否则编译器报错。

（4）out关键字：按引用传递参数，但是out限制参数在离开方法前必须进行赋值，否则编译器报错。
